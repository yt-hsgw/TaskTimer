# セキュリティポリシー

## 対象範囲

このプロジェクトは現在、設計と初期実装フェーズです。セキュリティレビューは、設計資料と今後の実装変更を対象にします。

## サポート対象

正式リリース前のため、サポート対象は `main` ブランチと最新のドラフトリリース候補のみです。

## 報告方法

秘密情報、個人のタスクデータ、データベースファイル、個人情報を含むスクリーンショットを公開Issueに貼らないでください。

公開後の脆弱性報告は、GitHubのPrivate Vulnerability ReportingまたはSecurity Advisoryを使ってください。利用できない場合は、公開Issueに機密情報を含めず、影響範囲と再現条件だけを最小限で記載してください。

公開Issueに貼ってはいけないもの:

- 実データを含むSQLiteファイル。
- タスク名、メモ本文、通知本文。
- 秘密情報、APIキー、トークン。
- 個人情報を含むスクリーンショット。
- OSユーザー名や端末固有パスが分かるログ。

## 基準

- MVPではアプリ実行時の外部通信を禁止する。
- ユーザー内容は端末内に保持する。
- ユーザー内容をログに出さない。
- 秘密情報をコミットしない。
- 新しい権限の追加には明示的な設計レビューを必要とする。

## 公開リポジトリ運用

- 公開前には `docs/public-readiness.md` を確認する。
- GitHub Secret scanningとDependabot alertsを有効にする。
- 依存関係更新PRは、Tauri権限、外部通信、ユーザー内容ログ出力への影響を確認してからマージする。
