# 063 ワークスペース表示切り替えとローカル検索を実装する

GitHub Issue: #157

## 背景

現在はリスト、タグ、カンバン、カレンダー、ポモドーロが左ペインで同列に並び、対象範囲と表示形式の責務が混在している。タスクを探す常設導線もない。

## 要件

- 左ペインはリスト、今日、お気に入り、設定へ限定する。
- 中央ペイン上部で `リスト`、`カンバン`、`カレンダー`、`タイムライン` を切り替える。
- 選択中の対象範囲と表示形式を別stateとして保持する。
- 右上へローカル検索入力とポモドーロアイコンを置く。
- 検索対象はタスク名、サブタスク名、メモ、タグ名とし、結果から詳細を開ける。
- 同じ表示形式の再選択では再取得しない。
- アプリ実行時の外部通信を追加しない。

## アーキテクチャ

Presentationは検索語、選択中範囲、表示形式、結果ポップアップだけを持つ。Applicationへ `SearchWorkItems` Queryを追加し、InfrastructureがSQLiteのページング外も含めて検索する。検索は更新トランザクションを開始しない読み取り境界とする。

## 設計理由

対象範囲と表示形式を直交させることで、選択中リストを保ったまま一覧と予定を比較できる。検索をReactの読込済み配列だけへ適用すると、大量データのページング外を検索できないため採用しない。

## トレードオフ

- `AppView` とUI設定の移行が必要になる。
- メモを含む部分一致検索は件数増加時に遅くなる。MVPは最大50件、200msデバウンス、入力120文字で制限し、必要になった時点でSQLite FTSを検討する。

## 代替案

左ペインの既存導線を残し、上部の表示切り替えも追加する。

不採用理由: 同じ機能への二重導線ができ、どちらが対象範囲を維持するのか分かりにくい。

## セキュリティと危険ケース

- SQLはバインド値を使い、`LIKE` ワイルドカードをエスケープする。
- 削除済みタスク、削除済みサブタスクを結果へ含めない。
- 検索語、タイトル、メモ、タグをログや外部へ送信しない。
- 古い非同期検索結果が新しい検索結果を上書きしないようrequest IDで破棄する。
- タグ別ビューを外してもタグデータを削除しない。

## 受け入れ条件

- 左ペインにタグ、カンバン、カレンダー、ポモドーロが表示されない。
- 中央上部で4表示を切り替え、対象範囲が維持される。
- ページング外のタスクとサブタスクを検索できる。
- 検索結果から対象詳細を開ける。
- 同じ表示形式を再選択しても不要な再取得とちらつきが起きない。
- TypeScript、Rust、SQLite検索、UI操作テストが成功する。
