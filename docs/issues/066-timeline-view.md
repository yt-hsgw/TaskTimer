# 066 タイムライン表示を実装する

GitHub Issue: #160

## 背景

開始予定と期限を持つ複数タスクを、リストやカレンダーとは別に期間比較できる表示がない。

## 要件

- 中央ペインの表示切り替えに `タイムライン` を追加する。
- 選択中の対象範囲を維持し、開始予定から期限までを横方向の期間バーで表示する。
- 日、週、月の粒度を切り替える。
- 期間未設定タスクは別の「日付未設定」領域へ表示する。
- バー選択で詳細を開く。期間変更D&Dは初回実装の対象外とする。

## 表示仕様

- `ListTaskPage` で読み込んだ親タスクを入力とし、選択中のリスト、今日、お気に入りのスコープとページング状態を維持する。
- 期間開始は `planned_start_date`、期間終了は `due_date` とする。
- 開始予定だけ、または期限だけを持つタスクは、その日を示す1日分のバーとして表示する。
- 開始予定と期限の両方がないタスクは、軸の下にある展開可能な「日付未設定」領域へ表示する。
- 日粒度は14日、週粒度は月曜始まりの12週、月粒度は12か月を固定表示する。前後移動は同じ表示単位数で行う。
- 期間が表示窓の端をまたぐ場合はバーを端で切り、継続していることを視覚的に示す。
- 表示窓と重ならない日付設定済みタスクは描画せず、前後移動で到達可能にする。
- タスク名列は横スクロール中も固定し、タイムライン内部だけを横・縦スクロール可能にする。
- タスクバーは所属リスト色を左端、タスク色または継承したリスト色を本文に使用する。色だけで状態を伝えない。
- バーまたは日付未設定タスクを選ぶと既存の詳細ペインを開き、同じタスクを再選択すると閉じる。
- 初回実装は読み取り専用とし、作成、移動、期間変更の操作は追加しない。

## Read Modelと境界

専用のRepository Queryや永続モデルは追加しない。既存の `ListTaskPage` が返す `TaskRowItem` とタスクツリーを、Presentationで次の表示用モデルへ射影する。

```mermaid
flowchart LR
  Scope["選択中スコープ"] --> Page["ListTaskPage\n最大200件"]
  Page --> Projection["Timeline projection\n開始・終了・未設定"]
  Projection --> Axis["固定長の日・週・月軸"]
  Axis --> Detail["既存タスク詳細"]
```

ページ末尾では既存の追加読み込みを使用する。これにより、一覧とかんばんと同じID集合、順序、総件数、カーソルを正とし、同じフィルターを別Queryで再実装しない。

## 設計理由

最初は読み取り専用として、カレンダーD&D Use Caseとの重複と誤更新を避ける。期間変更は利用状況を確認してから既存の予定移動Use Caseへ接続する。

タイムラインは時刻を配置するカレンダーではなく、開始予定と期限を比較する業務ビューである。既存のページング済みRead Modelを再利用すれば、更新トランザクションやDBマイグレーションを増やさず、安全に表示形式だけを追加できる。

## トレードオフ

- 初回は参照専用のため、その場で期間調整できない。
- 現在読み込まれたページだけを描画するため、未読ページの期間は追加読み込み前には表示されない。
- 固定長の軸は無制限の期間を一度に俯瞰できないが、描画量と操作位置を予測可能にできる。

## 代替案

カレンダーの週表示をタイムラインとして再利用する。

不採用理由: 時刻軸とプロジェクト期間軸では比較したい情報と密度が異なる。

専用の期間Queryを追加する案もある。表示窓内だけをDBで取得できる反面、リスト・今日・お気に入りのスコープ条件とページ境界が重複し、初回の読み取り専用ビューには変更量が大きいため採用しない。

## セキュリティと危険ケース

- 表示範囲の開始/終了を検証し、無制限な日付展開をしない。
- タイトルとメモをHTMLとして描画しない。
- 1回の取得を既存上限の200件に制限し、追加読み込みを使う。
- 外部通信や追加権限を導入しない。
- 日付文字列が不正な行は軸計算へ使用せず、画面全体を白画面にしない。
- 完了状態、色、日付を表示用に解釈するだけで、Repository更新や通知再登録を起こさない。

## 受け入れ条件

- 選択範囲の開始予定と期限を期間バーで比較できる。
- 開始予定だけ、期限だけ、両方を持つタスクがそれぞれ欠落せず表示される。
- 日付未設定タスクへ到達できる。
- 日、週、月を切り替え、前後の固定範囲へ移動できる。
- タスクバー選択で詳細を開き、粒度や期間の移動だけでは詳細を開かない。
- 追加読み込みで既存行を保持したまま次のページを表示できる。
- 大量データfixtureでスクロールと表示切り替えが破綻しない。
- キーボード操作と狭幅表示で主要情報へ到達できる。
