# 009: 左ナビゲーションとApp Shellを実装する

GitHub Issue: #26

## 目的

Microsoft To Do型の3ペイン構造を参考に、TaskTimerの左ナビ、中央ビュー、右詳細ペインの土台を作る。

## スコープ

- 左ペインのリスト一覧。
- お気に入りビュー導線。
- カレンダー導線。
- 最下部の設定導線。
- `Ctrl+B` による左ペイン開閉。
- 幅ごとのレイアウト切り替え。

## スコープ外

- タスク行の詳細デザイン。
- 右詳細ペインの全機能。
- カスタムリスト作成UI。

## 設計レビュー

### データモデル

`task_lists` とUI設定Read Modelに依存する。

実装では `list_task_lists` を左ペインのリスト一覧に使う。お気に入り導線の件数は現行の `list_tasks` 結果から `is_favorite` を集計し、タスク一覧Read Modelへの完全移行は #25 で扱う。

### トランザクション境界

左ペイン開閉状態を保存する場合は `UpdateUiPreference` で扱う。

#26ではUI設定更新commandが未実装のため、左ペイン開閉状態はReactのローカル状態として扱う。永続化は `UpdateUiPreference` 境界を追加するIssueで実装する。

### セキュリティ

外部フォント、リモート画像、分析SDKは追加しない。

ナビゲーションはローカルDBから取得したリスト名をReactのテキストとして描画する。HTML挿入はしない。

## トレードオフ

- 3ペインはデスクトップ効率が高いが、狭い幅ではドロワー制御が必要。
- 左ペインをアイコン化すると省スペースだが、初見の分かりやすさは落ちる。

## 代替案

現状の常時3カラムを維持する。

不採用理由:

- タスク一覧、カレンダー、設定の主従が曖昧なままになる。

## 受け入れ条件

- 左ペインからリスト、カレンダー、設定へ移動できる。
- `Ctrl+B` で左ペインを開閉できる。
- 768px未満でも主要ビューへ到達できる。

## レビュー結果

- 指摘事項: App Shellは左ナビと中央ビューの切り替えに限定し、タスク行刷新と右詳細ペイン刷新は #25 / #29 へ残す。
- 破綻シナリオ: 狭幅で左ペインが画面を塞ぐと主要ビューへ到達できないため、768px未満ではオーバーレイドロワーとして開閉する。
- スケール懸念: 左ペインはリスト一覧と件数のみを描画し、サブタスク全件の走査に依存しない。
- セキュリティ懸念: 外部アセット、外部通信、HTML描画を追加しない。
- テスト不足: 右詳細ペインやタスク行の最終デザインは別Issueで確認する。本Issueではビルドと主要ビュー到達性を確認する。
- 判断: フォローアップ付き承認。
