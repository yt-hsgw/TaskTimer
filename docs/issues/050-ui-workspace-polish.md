# 050 作業画面の操作配置と表示密度を整理する

## 背景

左ペイン、中央ペイン、詳細ペイン、設定画面に管理操作が分散し、日常的なタスク確認より設定操作が目立っている。
既存のデータモデルとApplication Use Caseを維持したまま、操作の置き場所と画面の重なり方を整理する。

## 対応範囲

- 左ペイン開閉ボタンを枠なしのアイコンボタンにする。
- 左ペインではタグの作成、名称変更、削除を行わず、タグ別ビューへの移動だけを扱う。
- リスト色の変更を左ペインからタスク詳細へ移す。色の保存単位は既存仕様どおりリスト単位とする。
- 設定アイコンを他のナビゲーションアイコンより判別しやすいサイズにする。
- 一覧またはかんばんで選択中の同じ親タスクをもう一度選ぶと、詳細を閉じる。
- かんばんカードにタスク一覧と同じ円形完了チェックを表示する。
- タスク詳細を中央ペイン上のオーバーレイとして表示し、背面の中央ビュー幅を変えない。
- タスク詳細で所属リストを直接選択できるようにする。
- タスク詳細でタグの作成、名称変更、削除、付け外しを扱う。
- カレンダーの上部と各セルの追加ボタンを廃止し、空きセルのダブルクリックで作成フォームを開く。
- 設定画面のデータ操作はJSON/CSVエクスポートだけを表示する。
- 通知失敗履歴を設定画面から外し、通知表示タイプをカード型ラジオ選択にする。

SQLiteバックアップ/復元Use Caseと通知失敗履歴Repositoryは削除しない。今回の要望は一般利用者向けUIの整理であり、既存の復元・診断データを破壊する変更は別判断とする。

## 画面責務

```mermaid
flowchart LR
  Left["左ペイン\n移動とリスト管理"]
  Center["中央ペイン\n一覧・かんばん・カレンダー・設定"]
  Detail["詳細オーバーレイ\nタスク固有の編集と関連設定"]

  Left --> Center
  Center --> Detail
  Detail --> Center
```

## 設計理由

- タグと色はタスクを見ながら判断する操作なので、詳細へ集約する。
- 詳細を別カラムにすると一覧幅が変化して情報が跳ねるため、中央ペインの上に重ねる。
- 所属リスト変更は既存 `UpdateTask`、リスト色変更は既存 `UpdateTaskList` を使い、Presentationに保存ロジックを複製しない。
- ダブルクリック追加は、タスク選択との競合を避けるため空きセルを契機にする。キーボードではフォーカスしたセルのEnterキーを代替操作にする。
- バックアップ/復元のApplication境界を残すことで、UI方針の変更だけで既存の安全な復元実装を失わない。

## トレードオフ

- 詳細オーバーレイは背面の右側を一時的に覆うが、詳細を閉じたときの一覧幅が変化せず、視線移動を抑えられる。
- タグ管理を詳細へ限定すると、タスク未選択時にタグだけを整理できない。日常作業の文脈を優先する。
- SQLiteバックアップ/復元のUIを外すと一般利用者はアプリ内復元できない。JSON/CSVは完全復元形式ではないため、将来再公開する場合は別の安全な導線を設計する。

## 代替案

詳細をモーダルダイアログとして中央に表示する。

不採用理由:

- 背面のタスク一覧やカレンダーとの関係が見えにくくなる。
- タスクを切り替えながら参照する日常操作に向かない。

## セキュリティ

- タグ名、タスク名、メモはReactのテキストとして描画し、HTMLとして解釈しない。
- 色は既存の許可済みトークンだけを扱い、任意CSS値を受け取らない。
- エクスポートには個人データが含まれる注意を維持する。
- 外部通信、追加OS権限、秘密情報は追加しない。

## 危険ケース

- 所属リスト変更後、現在のリスト表示対象からタスクが外れているのに古い詳細が残る。
- タグ削除時に、他タスクとの関連まで削除されることが利用者に伝わらない。
- カレンダー項目のダブルクリックが背面セルへ伝播し、作成フォームまで開く。
- 詳細オーバーレイが狭い画面で操作不能になる。
- JSON/CSVを完全復元用バックアップと誤認する。

## 受け入れ条件

- 左ペインにはタグとリスト色の編集操作が表示されない。
- タスク詳細から所属リスト、リスト色、タグを変更できる。
- 選択中の同じ親タスクを再選択すると詳細が閉じる。
- かんばんカードの完了チェックから既存の完了確認フローを実行できる。
- 詳細を開閉しても中央ビューの幅が変わらない。
- カレンダーの追加ボタンがなく、空きセルのダブルクリックまたはEnterで作成フォームが開く。
- 設定画面はJSON/CSVエクスポートだけを表示し、通知表示タイプはカード型ラジオ選択になる。
- TypeScriptビルドと主要画面の手動表示確認が成功する。
