# 069 今日ビューから開始予定日付きタスクを追加する

## 背景

共通タスク作成ダイアログは実リストとかんばんから開けるが、今日ビューでは追加ボタンを表示していない。また、現在の今日スコープは期限日だけを対象にするため、開始予定日を今日にして作成しても作成直後の一覧へ残らない。

## 要件

- 今日ビューのタスク件数右に親タスク追加ボタンを表示する。
- 今日ビューと今日ビュー上のかんばんから共通タスク作成ダイアログを開ける。
- 今日ビューからの通常作成は `planned_start_date` をローカル日付の今日にする。
- 作成先は最後に選択した実リストとし、今日というDBリストは作らない。
- 今日ビューは、親タスクまたはサブタスクの開始予定日/期限日が今日の親タスクを表示する。
- 今日件数と今日スコープのカレンダーも同じ条件を使う。
- 作成成功後に詳細を自動表示しない。

## 設計

`TaskCreatePreset.standard` に編集不可の `plannedStartDate` を追加する。App Shellは今日スコープの場合だけ今日の日付を設定し、共通ダイアログはその値を `CreateTask` へ渡す。ダイアログには自動設定される開始予定日を表示するが、通常リストからの作成では従来どおり表示しない。

InfrastructureのToday条件は、親タスクの `planned_start_date` / `due_date`、または未削除サブタスクの `planned_start_date` / `due_date` のいずれかが今日かを判定する。ページ本体、総件数、左ナビ件数、カレンダー、アクティブタイマーのスコープで同じ意味を保つ。

## トレードオフ

- 今日ビューの対象件数は、期限だけを数えていた従来より増える場合がある。
- 開始予定日は詳細UIの主要編集項目ではないが、今日から作成したタスクを今日の作業対象として維持するために既存フィールドを再利用する。

## 代替案

今日から作成したタスクの期限日も今日にする。

不採用理由: 開始予定と期限は別の業務意味を持ち、依頼されていない期限と期限通知を自動設定してしまう。

## セキュリティと危険ケース

- 今日の日付はPresentationで生成しても、Application層のISO日付検証を通す。
- 今日スコープのSQLは既存のバインド値を使い、文字列連結しない。
- 作成中に最後の実リストが削除された場合はApplication層で失敗させ、入力を保持する。
- タスク名、メモ、開始予定日をログまたは外部通信へ出さない。

## 受け入れ条件

- リスト、今日、かんばんから共通ダイアログを開ける。
- 今日から作成したタスクの `planned_start_date` が今日になる。
- 作成したタスクが今日ビューに残り、今日件数へ反映される。
- 開始予定日が今日のサブタスクを持つ親タスクも今日ビューに表示される。
- 期限通知は自動作成されない。
