# 011: 右詳細ペインを実装する

GitHub Issue: #29

## 目的

タスク選択時だけ右ペインを開き、サブタスク、期限、通知、タイマー、削除を一か所で操作できるようにする。

## スコープ

- タスク詳細表示。
- サブタスク追加。
- サブタスク期限設定。
- タスク期限設定。
- 通知設定。
- タイマー目標時間設定。
- タイマー開始/終了。
- タスク削除。

## スコープ外

- タイマー一時停止/再開。
- 繰り返し設定の永続化。
- ファイル添付。

## 設計レビュー

### データモデル

既存タスク/サブタスク/通知/タイマーに加え、タイマー目標時間が必要。

### トランザクション境界

- 期限更新と通知ルール更新は同一トランザクション。
- 削除は既存のソフト削除トランザクションを維持する。
- タイマー開始/終了は単一アクティブタイマー制約を維持する。

### セキュリティ

メモ本文をHTMLとして描画しない。通知本文にメモを含めない。

## トレードオフ

- 詳細ペインへ操作を集約すると便利だが、狭い幅では誤操作を避ける余白と確認が必要。
- サブタスク編集を同一ペイン内で行うと画面遷移は少ないが、状態管理が複雑になる。

## 代替案

詳細をモーダルで開く。

不採用理由:

- タスク一覧を見ながら作業しづらい。

## 受け入れ条件

- タスク選択時だけ右ペインが開く。
- 右ペインでサブタスク追加と期限設定ができる。
- 右ペインで通知表示とタイマー操作ができる。
- 削除時は確認が表示される。
