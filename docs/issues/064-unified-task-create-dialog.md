# 064 親タスク作成を共通ダイアログへ統合する

GitHub Issue: #158

## 背景

一覧はインラインフォーム、カレンダーは専用ポップアップを持ち、入力項目、キーボード操作、失敗時挙動が重複している。

## 要件

- 親タスク作成は全表示形式で共通ダイアログを使う。
- 一覧の追加ボタンと `Ctrl+N` は現在の対象リストを初期選択する。
- カレンダーのダブルクリックとドラッグ範囲作成は日付、時刻、期間を初期値にする。
- 保存成功後に閉じ、詳細を自動で開かない。
- 保存失敗時は入力内容と選択範囲を維持する。
- サブタスク追加はタスク詳細内のインラインフォームを維持する。

## 設計

App Shellが `TaskCreatePreset` とダイアログ開閉を所有する。各ビューは作成を直接保存せず、初期値だけをApp Shellへ通知する。保存は既存作成Use Caseへ集約し、カレンダー期間を持つ場合は予定付き作成Use Caseを使う。

`TaskCreatePreset` は次の2種類を持つ。

- `standard`: 所属リスト、期限日、期限時刻を初期値にし、`CreateTask` を使う。
- `scheduled`: 所属リスト、予定開始/終了、終日状態を初期値にし、`CreateScheduledTask` を使う。

両者はタイトル、所属リスト、メモ、保存/キャンセル操作を同じダイアログで共有する。予定付き作成は期限を自動設定せず、通常作成は予定期間を自動設定しない。Presentationで作成後に追加更新する方式は、途中失敗でタスクだけが残るため採用しない。

ダイアログはApp Shell直下に置き、表示形式の子コンポーネントを切り替えても入力stateを失わない。モーダル表示中はポインター操作と `Ctrl+B` を含む背面操作を遮断し、最初にタイトルへフォーカスする。`Tab` / `Shift+Tab` はダイアログ内を循環し、閉じた後は起点へフォーカスを戻す。

`Escape` と背景クリックは未入力なら閉じる。入力済みの場合は破棄確認を表示する。明示的なキャンセルと閉じるボタンはそのまま閉じる。保存中は閉じる操作と二重送信を無効にし、失敗時は入力とカレンダー選択範囲を保持してエラーをダイアログ内へ表示する。

## トレードオフ

- 各ビューのローカルフォームstateを移行する必要がある。
- ダイアログは背面情報を覆うが、入力責務とエラー表示を統一できる。

## 代替案

各ビューのフォームを見た目だけ同じコンポーネントへする。

不採用理由: 開閉、フォーカス、保存、エラーstateが各ビューへ残り、挙動差が再発する。

## セキュリティと危険ケース

- タイトル、メモ、日付、時刻、期間、リストIDをApplication層でも再検証する。
- フォーカストラップ、Escape、背景クリック、送信中の二重保存を扱う。
- 表示モード切り替え時に入力済みダイアログを黙って破棄しない。
- ダイアログ表示中に `Ctrl+N` が再入力されても、作成stateを初期化しない。
- 作成中に選択リストが削除された場合は、Applicationの存在確認で保存を失敗させ、入力を保持する。
- ユーザー入力をHTML描画またはログ出力しない。

## 受け入れ条件

- 実装済みの一覧、カンバン、カレンダーの親タスク作成が同じダイアログを使い、タイムライン追加時も同じプリセット境界を再利用できる。
- `Ctrl+N`、Escape、フォーカス移動、二重送信防止が動作する。
- カレンダーの範囲初期値と保存後の表示更新が維持される。
- 保存失敗時に入力内容が維持され、保存成功時だけ閉じる。
- サブタスク追加の既存UXに回帰がない。
