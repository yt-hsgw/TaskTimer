# 027: タスク詳細とサブタスク選択UXを改善する

GitHub Issue: #68

## 目的

右詳細ペインを初期表示では参照中心にし、期限・目標時間・繰り返し・サブタスク選択を業務中に迷わず操作できる形へ整える。

## スコープ

- ビュー切り替え時は右詳細ペインを閉じる。
- タスク/サブタスク詳細の初期表示は現在情報の参照を主にする。
- 開始日は詳細UIから外し、期限日と期限時刻を設定対象にする。
- 期限は `今日`、`明日`、`時間設定`、削除アイコンを用意する。
- `時間設定` はポップアップで期限日と期限時刻を設定する。
- 期限操作は添付参考に合わせて小さなチップUIにし、期限設定済みの場合は選択済みチップだけを表示する。期限解除はチップ内の削除アイコンで扱い、解除後に `今日`、`明日`、`時間設定` を復帰させる。
- タイマー目標時間はプリセット選択と手入力の両方を可能にする。
- 繰り返しは通常無効の最小表示にし、有効時だけ頻度と間隔を展開する。
- 編集セクションは「基本情報」ではなく、対象に応じて「タスクを編集」「サブタスクを編集」と表示する。
- 親タスク詳細ではサブタスク一覧と追加だけを扱い、既存サブタスクの編集フォームは表示しない。サブタスク追加フォームは「サブタスクの追加」押下後に表示する。
- タスク一覧でサブタスクを展開でき、サブタスク選択で右詳細ペインにサブタスク詳細を表示する。
- サブタスク詳細から親タスク詳細へ戻れる。
- サブタスクの完了済みチェックは親タスクと同じく未完了へ戻せる。

## スコープ外

- 開始予定日の完全削除マイグレーション。既存データとの互換性のためDBカラムは残す。
- 繰り返しタスク完了時の次回タスク自動生成。
- ドラッグ&ドロップによるサブタスク並び替え。

## 実装方針

- `due_time` をタスク/サブタスクに追加し、期限時刻の正とする。
- 期限通知の `notify_at` は `due_date + due_time` から生成する。時刻未設定の場合は従来通り日付の開始時刻を使う。
- カレンダーRead Modelは期限項目の `time` に `due_time` を返す。
- Presentationは `selectedTaskId` と `selectedSubtaskId` を分け、親タスク詳細とサブタスク詳細を同じ右ペインで切り替える。
- サブタスク一覧展開は既存の `TaskWithSubtasks` を使い、追加のDBクエリを増やさない。
- 詳細ペイン内の開閉状態は、対象IDが変わる時だけ初期化する。値更新後のスナップショット再取得ではセクション状態を維持し、画面全体のちらつきを抑える。

## 設計レビュー

### データモデル

`planned_start_date` は互換性のため保持するが、詳細UIでは編集対象にしない。期限時刻は `due_time` として分離し、日付なしの時刻だけは許可しない。

### トランザクション境界

- タスク/サブタスク更新、期限時刻更新、通知ルール同期、繰り返し同期は既存の詳細更新トランザクションに含める。
- 右ペインの表示切替はPresentation状態のみで、DB副作用を持たない。

### セキュリティ

タイトル、メモ、期限日、期限時刻、目標時間、繰り返し入力はApplication境界で検証する。メモ本文をHTMLとして描画しない。外部通信や追加Tauri権限は追加しない。

### 破綻シナリオ

- 期限時刻だけ保存できると通知・カレンダーの基準日がなくなる。
- 親タスク詳細内にサブタスク編集フォームを残すと、親と子の編集責務が混ざる。
- ビュー切り替え後に古い詳細が残ると、別ビューの対象を編集してしまう。
- 一覧展開で全サブタスク編集フォームを描画すると、タスク数が多い場合に描画負荷が増える。
- 削除後に詳細ペインの選択状態が残ると、存在しない対象を操作しているように見える。
- OS確認ダイアログに依存すると環境によって削除操作が届かない可能性があるため、詳細ペイン内の2段階確認で削除する。

### スケール

タスク一覧の展開は選択中のタスク配下だけを描画し、全サブタスクにフォーム状態を持たせない。詳細ペインは選択対象だけのフォーム状態を持つ。

## トレードオフ

- `due_time` を追加するとマイグレーション範囲は増えるが、時間設定を通知・カレンダーと一貫させられる。
- 開始予定日カラムを残すとモデルは少し冗長になるが、既存データと通知ルールの互換性を保てる。
- 参照中心の詳細は誤編集を減らす一方、タイトルやメモを編集するには明示的に編集セクションを開く必要がある。

## 代替案

期限時刻をUIだけに表示し、DBには保存しない。

不採用理由:

- 通知時刻とカレンダー表示に反映できず、業務利用では期待と実データがずれる。

## 受け入れ条件

- ビューを切り替えると右詳細ペインが閉じる。
- タスク/サブタスク詳細の初期表示が参照中心で、編集フォームが常時露出しない。
- 期限未設定時は今日、明日、期限日時ポップアップを表示し、期限設定済み時は選択済みチップと削除アイコンだけを表示する。
- 期限時刻が通知とカレンダーの時刻表示へ反映される。
- 目標時間をプリセットまたは手入力で保存できる。
- 繰り返し入力は有効時だけ展開される。
- 編集セクション名から編集画面であることが分かる。
- 親タスク詳細ではサブタスク一覧と追加だけを扱い、既存サブタスク編集はサブタスク詳細で行う。
- サブタスク追加フォームは追加ボタン押下後だけ表示される。
- タスク一覧でサブタスクを展開し、サブタスク選択後に親タスクへ戻れる。
- サブタスク完了済みチェックを押すと未完了に戻せる。

- 開始予定日の完全な廃止は別Issueでデータ移行を検討する。
