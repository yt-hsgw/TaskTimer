# [Domain] タイマー・削除・完了の境界ケースを固める

## 目的

タイマー開始中の削除、未完了サブタスク付き親タスク完了、ソフト削除履歴の扱いを安全にする。

## 対応内容

- 親タスク完了時の確認フローを実装する。
- タスク削除時の子サブタスク/タイマー/通知ソフト削除を実装する。
- サブタスク削除時のタイマー/通知ソフト削除を実装する。
- タイマー開始中の対象削除時の挙動を定義・実装する。
- ソフト削除済みデータを通常クエリから除外する。

## 完了条件

- 未完了サブタスクがある親タスクは確認後にだけ完了できる。
- タスク削除で関連履歴がソフト削除される。
- タイマー開始中の対象削除で状態不整合が起きない。
- 境界ケースのテストがある。

## 危険ケース

- アクティブタイマーが削除済み対象を指し続ける。
- タイマー履歴が通常画面から見え続ける。
- 親タスク完了でサブタスク状態を誤って変更する。

## ラベル候補

- `enhancement`
- `domain`
- `priority: P1`
