# 067 詳細画面と左ペインの管理操作を整理する

GitHub Issue: #161

## 対応範囲

- タスク詳細のタイマー、通知セクションを外す。
- タスク詳細のリスト色編集を外し、所属リスト選択だけを残す。
- 左ペインのリスト編集へリスト色選択を移す。既定リストも色だけ変更できる。
- タグ一覧を左ペインから一旦非表示にする。タグデータと詳細での管理は維持する。
- かんばん状態名はフォーカスが外れた時も保存する。

カンバン、カレンダー、ポモドーロの左ペイン導線は、代替導線を実装するIssue 063まで残す。先に削除して機能へ到達できなくなる回帰を避ける。

## 設計理由

詳細はタスク固有情報へ限定し、リスト固有の色はリスト編集へ戻す。通常タイマー操作は一覧とサブタスク行、通知設定は設定画面へ責務を集約する。

## トレードオフ

- タスク詳細から通常タイマーを操作できなくなるが、同じ操作が一覧行へ残る。
- タグ別ビューは一時的に到達不能になるが、データを削除せず検索と将来の絞り込みへ残す。

## セキュリティと危険ケース

- 色は許可済みトークンだけを既存Use Caseへ渡す。
- フォーカス外保存とEnter送信が重複しないよう単一の保存関数へ集約する。
- 空の状態名は保存せず元の値へ戻す。
- 左ペインからタグを外しても関連行を削除しない。

## 受け入れ条件

- タスク詳細にタイマー、通知、リスト色編集が表示されない。
- 左ペインで各リスト色を変更できる。
- 左ペインにタグ一覧が表示されない。
- かんばん状態名がEnterとフォーカス外のどちらでも一度だけ保存される。
- 既存の一覧タイマー、サブタスクタイマー、タグ管理に回帰がない。

## 検証結果

- TypeScript型検査とVite本番ビルドに成功した。
- Rust全106テストに成功した。
- 50タスク、200サブタスクのUIスモークで、リスト色更新、blur保存1回、詳細セクション非表示、既存タイマー操作を確認した。
- 実行時プライバシー監査とREADMEスクリーンショット再生成に成功した。
