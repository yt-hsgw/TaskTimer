# 068 左ペインのリスト管理操作を縦三点メニューへ集約する

GitHub Issue: #163

## 背景

左ペインでは各リストの編集ボタンと削除ボタンが常時表示され、移動先を選ぶ日常操作より管理操作が目立っている。ボタン分の領域によってリスト行の高さと密度も不安定になる。

## 要件

- 常時表示している編集ボタンと削除ボタンを廃止する。
- リスト行のタスク件数の右側へ縦三点アイコンを配置する。
- 縦三点アイコン押下時は、その行にアンカーしたポップアップで `編集` と `削除` を選べる。
- 編集ではリスト名とリスト色を変更する。既定リストは名称を固定し、色だけ変更できる。
- 編集領域からフォーカスが外れた時、または別の場所をクリックした時に変更を保存する。保存ボタンと取消ボタンは表示しない。
- カスタムリスト名が空白だけの場合は更新せず、編集前の名称へ戻す。色が変更されている場合は、元の名称と変更後の色を保存する。
- 削除はカスタムリストだけに表示し、既定リストでは表示しない。
- ポップアップは外側クリック、`Escape`、別リスト選択、左ペイン折りたたみで閉じる。
- 同時に開けるポップアップは1件だけとする。
- キーボード操作とフォーカス表示を提供する。

## 設計理由

リスト行は移動操作を主責務とし、低頻度の管理操作をコンテキストメニューへ退避する。件数と管理メニューを同じ末尾領域へ置き、メニュー開閉で行の幅や高さを変えない。

## トレードオフ

- 編集と削除は1クリック増える。
- ポップアップのフォーカス管理と狭幅時の配置調整が必要になる。
- フォーカスアウト保存では、色選択など編集領域内のフォーカス移動と領域外への移動を区別する必要がある。

## 代替案

リスト行を右クリックして管理メニューを開く。

不採用理由: 右クリックは発見しにくく、タッチ操作とキーボード操作の代替導線も別途必要になる。

## セキュリティと危険ケース

- リスト名は既存Application Use Caseで検証し、HTMLとして描画しない。
- 削除確認と既定リスト削除禁止をPresentationだけへ依存させず、Applicationの制約を維持する。
- ポップアップが画面外へはみ出す場合は上下方向を反転する。
- 長いリスト名や3桁以上の件数でも縦三点アイコンを押し出さない。
- 空白の名称をApplication Use Caseへ渡さず、既存名称へ戻す。更新失敗時は編集状態を維持する。

## 受け入れ条件

- リスト行に編集・削除ボタンが常時表示されない。
- タスク件数の右側に縦三点アイコンが表示される。
- ポップアップから編集とカスタムリスト削除を実行できる。
- 既定リストでは削除を選択できない。
- 編集領域外へフォーカスを移すと保存され、保存ボタンと取消ボタンが表示されない。
- 空白の名称で編集を終えると編集前の名称へ戻り、入力検証エラーを発生させない。
- メニュー開閉でリスト行が移動せず、マウスとキーボードで操作できる。
- UIテストと既存のリスト管理テストが成功する。

## 実装結果

- リスト件数の右側へ縦三点トリガーを固定し、常時表示していた編集・削除ボタンを廃止した。
- メニューは `document.body` へポータル表示し、画面下端では上側へ反転する。リサイズと左ペインスクロール時は閉じる。
- 外側クリック、`Escape`、別ビュー選択、左ペイン折りたたみで閉じ、`Escape` ではトリガーへフォーカスを戻す。
- 既定リストは編集だけ、カスタムリストは編集と削除を表示する。編集は領域外へのフォーカス移動で自動保存し、空白名称は元の名称へ戻す。
- 更新・削除は既存Application Use Caseを利用し、更新失敗時は編集状態を維持する。
- 大規模データUI計測で表示内容、フォーカス復帰、画面内配置、トリガーへのアンカー、色編集を自動確認する。
