# 053: タスク一覧の200件上限をページングで解消する

GitHub Issue: #131

## 背景

`list_tasks` と `list_task_rows` は1回200件を上限とするが、追加読み込み境界がない。大量DBでも初期描画は抑えられる一方、201件目以降へ到達できない。

## 要件

- タスクツリーとタスク行Read Modelへ安定したカーソルページングを追加する。
- タスク、今日、お気に入り、タグ、かんばんのスコープをDB側で絞る。
- 追加読み込み時に選択、展開、スクロール位置を維持する。
- DB集計件数と読み込み済み件数を区別する。

## 設計方針

### Application DTO

`ListTaskPage` は次を1レスポンスで返す。

- 同じタスク集合から組み立てたタスク詳細用ツリーと一覧用 `TaskRow`。
- DB集計によるスコープ内総件数。
- 次ページがある場合だけ返すカーソル。
- 左ペイン用の今日未完了件数とお気に入り件数。

スコープは `list`、`today`、`favorites`、`tag`、`board` の判別共用体とする。ページ上限は200件とし、0件、201件以上、空の識別子、不正な日付、構成要素が欠けたカーソルをApplication境界で拒否する。

### カーソル

並び順とカーソルは次の組を使う。

1. 完了区分。未完了を0、完了を1とする。
2. `sort_order`。
3. `created_at`。
4. `id`。

Infrastructureは同じ比較順序でカーソルより後の行を取得し、上限より1件多く読んで次ページ有無を判定する。同順位・同一作成日時でも`id`で順序を確定する。

### Repository境界

Repositoryは1回の接続ロック内でページ対象タスクを確定し、そのID集合からタスクツリーと `TaskRow` を作る。これにより別々のクエリ結果がずれて詳細を開けない状態を防ぐ。状態変更Use Caseのトランザクション境界と本番テーブル列は変更しない。カーソル順の式評価を全件ソートへ退化させないため、論理削除・アーカイブを除いたタスクへページ順専用インデックスを追加する。

### Presentation状態

- 初期表示は200件を読み込む。
- 「さらに読み込む」は既存配列へ次ページだけを追記し、既存オブジェクト、選択、サブタスク展開、スクロール位置を維持する。
- ビュー切替は新しいスコープの先頭ページを読み込む。
- 完了、削除、移動など並び順を変え得る更新後は、現在の読み込み件数に達するまで先頭から再取得する。古いカーソルを再利用しない。
- 合計件数はDB集計値、読み込み済み件数は配列長として別表示する。

### セキュリティ・権限

- スコープ種別と識別子を検証し、SQL断片をPresentationから受け取らない。
- カーソルをログへ出さない。タスク名、メモ、タグ名も計測ログへ出さない。
- 外部通信、新しいTauri capability、OS権限を追加しない。

## 代替案

仮想リストだけを追加する。

不採用理由: DOM数は減るが、Repositoryが返さない201件目以降へ到達できない問題は解消しない。

## 危険ケース

- 同順位タスクがページ間で重複または欠落する。
- ページ読み込み中の完了・削除でカーソルがずれる。
- 今日やお気に入りをPresentationの全件走査で実装する。
- タスク行と詳細用親子データが不整合になる。
- ビュー切替直前の追加読み込み応答が、新しいビューへ混ざる。
- 追加読み込みのたびに既存行を置き換え、選択・展開・スクロールが失われる。
- 完了区分をカーソルに含めず、未完了と完了の境界で重複する。

## 受け入れ条件

- 5,000件DBで201件目以降へ到達できる。
- ページ間の重複・欠落がないRepositoryテストがある。
- 主要スコープで対象外データを全件取得しない。
- 不正なスコープ、上限、カーソルをApplication境界で拒否する。
- 一覧とかんばんで追加読み込みでき、DB総件数と読み込み済み件数を区別できる。
- 追加読み込み後も選択、展開、スクロール位置を維持する。
- 外部通信、新しい権限、ユーザーデータのログ出力を追加しない。

## 実装結果

- `ListTaskPage` のApplication DTO、Repository境界、Tauri commandを追加した。
- 一覧、今日、お気に入り、タグ、かんばんをDBスコープへ移し、初期200件と追加読み込みを実装した。
- 5,000件の全ページ巡回と、401件のPresentation追加読み込みを性能ゲートへ追加した。
