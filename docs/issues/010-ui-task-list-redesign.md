# 010: タスク一覧を新UIへ置き換える

GitHub Issue: #25

## 目的

中央ペインを主作業領域にし、タスクの状態、進捗、期限、お気に入りを一覧で判断できるようにする。

## スコープ

- 円形チェックボタン。
- タスクタイトル。
- 右端のお気に入りボタン。
- サブタスク進捗バーと `1/2` 表示。
- 期限ありアイコン。
- 完了セクション。
- 完了タスクの訂正線と半透明表示。
- `Ctrl+N` と最下部追加ボタン。

## スコープ外

- 右詳細ペインの詳細編集。
- 繰り返し設定。
- カレンダー移管。

## 設計レビュー

### データモデル

タスク状態、お気に入り、サブタスク進捗Read Modelに依存する。完了セクションは `status` と `completed_at` を正とし、追加フィールドは不要。

実装では一覧描画に `TaskRowDto` を使う。詳細操作や親タスク完了時の未完了サブタスク確認は既存の `TaskWithSubtasks` を併用する。

表示ルール:

- 未完了タスクは上部、完了タスクは下部の完了セクションへ表示する。
- サブタスクがある親タスクだけ、`completedSubtaskCount / subtaskTotalCount` と進捗バーを表示する。
- 期限日は `dueDate` がある場合だけアイコンと日付を表示する。
- タスクまたは子サブタスクが実行中の場合、行に実行中インジケーターを表示する。
- 一覧行にメモ本文は表示しない。

### トランザクション境界

- 完了は既存 `CompleteTask` / `CompleteSubtask` を利用する。
- お気に入りは `ToggleTaskFavorite` を追加する。
- `ToggleTaskFavorite` はタスク1件の `is_favorite` と `updated_at` だけを更新する。
- `Ctrl+N` と最下部追加ボタンは、選択中リストに対する作成フォームを開く。現時点の作成Use Caseは初期リスト固定のため、カスタムリスト作成/所属リスト指定は別Issueで扱う。

### セキュリティ

タスクタイトルはReactの通常テキストとして描画し、HTMLとして描画しない。

お気に入り切り替えcommandはタスクIDを検証し、SQLプレースホルダで更新する。外部通信や外部アセットは追加しない。

## トレードオフ

- 情報量を増やすと行が混みやすい。進捗バーはサブタスクがある親タスクだけに限定する。
- 完了セクションは便利だが、完了タスクが大量にある場合は折りたたみと上限表示が必要。

## 代替案

カード型の大きなタスク表示にする。

不採用理由:

- 実務で大量タスクを眺めるには密度が低い。

## 受け入れ条件

- タスク行に円形チェック、タイトル、お気に入り、期限状態が表示される。
- サブタスクがある親タスクだけ進捗バーが表示される。
- 完了タスクが下部の完了セクションへ移動する。
- `Ctrl+N` でタスク追加を開始できる。

## レビュー結果

- 指摘事項: タスク一覧刷新は中央ビューに限定し、右詳細ペイン移管は #29 へ残す。
- 破綻シナリオ: 完了済みタスクが大量になると一覧が伸びるため、完了セクションは折りたたみ可能にする。
- スケール懸念: 一覧行は `TaskRowDto` を使い、サブタスク全件をPresentationで走査しない。
- セキュリティ懸念: タイトルはテキスト描画のみ。お気に入り更新はID検証とRepository境界を通す。
- テスト不足: 実機TauriでのDB連携UI確認は継続課題。本IssueではRustテスト、frontend build、ブラウザでの表示/ショートカット確認を行う。
- 判断: フォローアップ付き承認。
