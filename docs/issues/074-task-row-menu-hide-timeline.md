# 074 タスク行三点メニューとタイムライン一時非表示を整理する

GitHub Issue: #188

## 背景

タスク一覧の行から期限設定、サブタスク追加、削除、リスト選択、上下移動へ到達する導線が不足している。詳細ペインを開かずに小さな補助操作を行えるようにする一方、タイムラインは改善余地が大きいため通常の表示切り替えから一旦隠す。

## スコープ

- 中央ペインの表示切り替えからタイムラインを非表示にする。
- タスクリストの各タスク行に縦三点メニューを追加する。
- メニューから次の操作に到達できるようにする。
  - 期限の設定
  - サブタスク追加
  - 削除
  - リスト選択
  - 上に移動
  - 下に移動
- 期限設定は小さなポップアップで入力する。
- サブタスク追加は通常タスク追加と同じ作成ダイアログを使う。
- リスト選択はドロップダウンではなく、ポップアップ内にリスト候補を並べる。
- リスト作成数は初期リストを含めて最大10件に制限する。

## 設計

### タイムライン

既存の `timeline` コード、Read Model、UI設定値は削除しない。中央ペイン上部の表示切り替えだけから除外し、保存済みUI設定が `timeline` の場合は `list` へフォールバックする。これにより後続の再設計で再利用しやすくしつつ、一般操作では未成熟なビューへ遷移しない。

### タスク行メニュー

三点メニューはPresentationの補助操作として扱う。チェック、スター、タイマー、行選択とイベントを分離し、メニュー操作では右ペインを開かない。

期限設定とリスト選択は既存Use Caseを再利用する。リスト選択は候補をボタン一覧として表示し、選択時に即時保存する。入力値はPresentationで空白を正規化し、Application側の既存検証で最終防衛する。

サブタスク追加は通常タスク追加ダイアログへ `subtask` プリセットを追加して共通化する。親タスクIDと親タスク名はApp Shellが解決し、ダイアログはタイトル、期限、メモだけを入力する。サブタスクは親タスクに属するため、作成ダイアログ上のリスト選択は表示しない。

### 上下移動

上下移動は新しいApplication Use Caseとして `ReorderTaskWithinList` を追加する。対象タスクの所属リスト内で、未削除かつ非アーカイブの隣接タスクと `sort_order` を入れ替える。今日、お気に入りなどのスマート範囲で押した場合も、表示中スコープではなく所属リスト内の順序を更新する。

## トランザクション境界

- `UpdateTask`、`CreateSubtask`、`DeleteTask` は既存境界を使う。
- `ReorderTaskWithinList` は1トランザクションで対象タスクと隣接タスクを取得し、両者の `sort_order` を入れ替える。
- `CreateTaskList` は初期リストを含むアクティブリスト件数を同一トランザクションで確認し、10件以上なら作成しない。
- 境界行で隣接タスクが存在しない場合は更新せず成功扱いにする。

## 代替案

上/下移動をフロントの表示配列だけで入れ替える案もある。しかし再読み込みやページングで順序が戻り、DBの `sort_order` とUIが食い違うため採用しない。

タスク行メニューを右ペイン操作へのショートカットにする案もある。しかし「詳細を開かずに補助操作を完了する」目的とずれるため、今回はポップアップ内で完結させる。

サブタスク追加専用の小型フォームを行メニュー内に残す案もある。しかし通常タスク追加と入力項目、フォーカス制御、エラー表示が分岐し続けるため、共通ダイアログへ寄せる。

## 危険ケース

- メニュー押下が行選択やD&D開始として扱われる。
- スマート範囲で上下移動した際、表示順と保存順の意味が混同される。
- 期限設定ポップアップの空日付と時刻だけ指定がApplication検証をすり抜ける。
- サブタスク作成ダイアログが親タスクの選択状態を暗黙に参照し、別タスクへ作成してしまう。
- リスト上限をUIだけで判定し、コマンド経由で11件目を作成できる。
- タイムライン非表示後も保存済み設定で空画面に遷移する。

## セキュリティ

- タスク名、メモ、サブタスク名をHTMLとして描画しない。
- 新しい外部通信、権限、ログ出力は追加しない。
- 削除は既存のソフト削除方針を維持する。

## 受け入れ条件

- 通常の表示切り替えからタイムラインを選択できない。
- タスク行の縦三点メニューから期限設定、サブタスク追加、削除、リスト選択、上/下移動を実行できる。
- サブタスク追加は通常タスク追加と同じダイアログで入力できる。
- リスト選択はドロップダウンではなく、ポップアップ内のリスト候補から選択できる。
- リストは初期リストを含めて10件を超えて作成できない。
- メニュー、チェック、スター、タイマー操作で右ペインが意図せず開かない。
- 上/下移動は境界行で破綻しない。
- 削除は既存のソフト削除方針を維持する。
