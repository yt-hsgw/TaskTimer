# MVP仕様

## 目的

Windows/macOSで動作するオフライン対応デスクトップタスクタイマーを作る。タスク計画、サブタスク計画、単一アクティブタイマー、ローカル通知、カレンダー、かんばんをMVP範囲とする。

## ユーザー

主なユーザー:

- 自分の業務タスクを整理し、実作業時間をローカルで記録したい個人の実務者。

MVPで扱わないこと:

- チーム共同編集。
- クラウド同期。
- モバイルアプリ。
- 外部カレンダー同期。
- Webログイン。
- 分析送信。
- アプリ実行時のネットワーク通信。

## 機能要件

### タスク

- ユーザーはタスクを作成、編集、完了、削除できる。
- タスクはタイトル、状態、開始予定日、期限日、期限時刻、メモ、並び順、作成日時、更新日時を持つ。
- タスクは必ず1つのタスクリストに所属する。
- タスクタイトルは必須。
- タスク期限日は開始予定日より前にできない。
- 完了したタスクは完了日時を記録する。
- 未完了サブタスクがある親タスクも、明示確認後であれば完了できる。
- 初期MVPの `todo`、`in_progress`、`done` は後方互換値として保持する。拡張後は業務状態をカスタム列、完了状態をライフサイクルとして別々に管理する。
- タスク削除時は、タスク、子サブタスク、タイマー履歴、ポモドーロ履歴、通知ルールを同一トランザクションでソフト削除する。
- アーカイブ状態はドメインモデルに残すが、ユーザー操作としてのアーカイブ/復元はMVP後の #52 で扱う。

### タスクリスト

- 初期リストとして `タスク` を持つ。
- ユーザーはカスタムリストを作成、名称変更、削除できる。
- 初期リストは名称変更、削除できない。
- リスト名は必須、最大80文字、アクティブなリスト間で一意とする。
- 選択中リストで作成したタスクは、そのリストに所属する。
- タスク詳細から所属リストを変更できる。
- カスタムリスト削除時、所属タスクは初期リストへ移動し、タスク、サブタスク、タイマー履歴、通知ルールは削除しない。
- リスト色は左ペインのリスト編集で変更し、`green`、`blue`、`amber`、`rose`、`violet`、`gray` の許可済みトークンだけを保存する。

### タスク表示色

- 親タスクは所属リストとは独立した表示色を任意で持てる。
- タスク表示色が未設定の場合は、所属リスト色を継承する。
- サブタスクは個別色を持たず、親タスクの実効表示色を継承する。
- タスクを別リストへ移動しても、明示設定したタスク表示色は保持する。
- タスク表示色は親タスク詳細で変更し、許可済み色トークンだけを保存する。
- JSON/CSVエクスポートでは、未設定による継承と明示設定を区別する。

### サブタスク

- ユーザーはタスク配下のサブタスクを作成、編集、完了、削除できる。
- サブタスクはタイトル、状態、開始予定日、期限日、期限時刻、メモ、並び順、作成日時、更新日時を持つ。
- サブタスクタイトルは必須。
- サブタスク期限日は開始予定日より前にできない。
- サブタスクは親タスクとは独立したタイマー履歴を持てる。
- サブタスク削除時は、サブタスク、タイマー履歴、ポモドーロ履歴、通知ルールを同一トランザクションでソフト削除する。
- アーカイブ状態はドメインモデルに残すが、ユーザー操作としてのアーカイブ/復元はMVP後の #52 で扱う。

### タイマー

- ユーザーは1つのタスクまたは1つのサブタスクに対してタイマーを開始できる。
- アプリ全体で同時に開始できる通常タイマーまたはポモドーロは1件だけ。
- ユーザーはアクティブタイマーを停止できる。
- タイマー履歴は保持する。
- タイマー開始中にアプリを閉じた場合、再起動時は保存済みの `started_at` から経過時間を算出する。
- 完了済みまたはアーカイブ済みの対象はタイマー開始不可。
- ポモドーロモードは #107 で通常タイマーとは別の任意モードとして追加する。Application境界、SQLite保存、通常タイマーとの相互排他、作業完了、休憩開始/完了、休憩スキップ、キャンセル、一時停止/再開のフェーズ遷移、設定画面での既定値編集、右ペインでの操作UI、期限到達同期、作業/休憩終了通知、自動開始設定の実行処理まで追加済み。休憩時間は作業タイマー履歴へ含めず、完了済みまたはアーカイブ済み対象への次作業自動開始は行わない。

### 通知

- ユーザーはタスクまたはサブタスクの開始予定日・期限に対してローカル通知を設定できる。
- 通知設定はローカルに保存する。
- デフォルトの通知表示モードは対象タイトルのみを表示する。
- ユーザーは設定画面で通知全体ON/OFFと通知表示モードを変更できる。
- 通知表示モードは `title_only` と `generic` をサポートする。
- 通知全体OFF中は、開始予定日・期限日の通知ルールを保持したままOS通知送信を行わない。
- OS通知登録はDBコミット後に実行する。
- OS通知登録に失敗した場合、アプリは再試行可能な登録状態を記録する。
- MVPでは、アプリ起動中または再読み込み時に期限到来済みの通知ルールをOS通知として送信する。
- #51では、将来時刻通知を段階導入にする。第1段階ではアプリ起動中にローカルスケジューラが次回通知時刻まで待機し、既存の期限到来dispatchを呼ぶ。
- ポモドーロ作業/休憩終了通知は、アプリ起動中、復帰時、表示更新時の期限到達同期で送信する。
- アプリ完全終了中の通知保証は、Windows先行PoC #123 で採用可否を判断する。macOSは署名・公証準備まで後回しにする。

### カレンダー

- 初期MVPのカレンダー画面は週表示とした。
- 現在のカレンダー画面は週/日/月を切り替えられる。
- 週/日表示には、開始予定日、期限日、期限時刻、タイマー実行中の対象を表示する。
- 日付のみの開始予定/期限は、時間指定予定と誤認しないよう「日付のみ」行へ表示する。
- 月表示は月曜始まりのグリッドで表示し、1日あたりの表示件数を制限する。残件数ボタンから、その日の表示対象予定を一覧表示できる。
- カレンダー項目のサブタスクには親タスク名を表示する。
- カレンダー項目の左端は所属リスト色、本文は親タスクの実効表示色で表示する。
- 色だけに意味を持たせず、タイトル、親タスク名、予定種別のテキストを維持する。
- 週/日/月表示は前後へ移動できる。
- タスクとサブタスクは期限とは独立した予定開始/終了を持てる。
- 予定期間は週/日表示では時間ブロック、月表示では日単位ブロックとして表示する。
- 予定期間の開始端/終了端はポインターまたはキーボードで調整できる。
- 時刻あり期間は15分刻み、終日期間は1日刻みで調整する。
- 予定期間の変更は期限と期限通知を変更しない。
- カレンダーの時間帯/日付をダブルクリックして作るタスクは、初期予定期間を持つ。

### UI/UX改修

- アプリは左ペイン、中央ペイン、右ペインの3ペイン構成を基本とする。
- 左ペインには今日、お気に入り、リスト一覧の順で表示し、最下部に設定画面導線を置く。
- 左ペインではカスタムリストの作成、名称変更、削除を行える。
- リスト一覧は展開可能とし、リスト追加は一覧の下部から行う。
- 左ペインのリスト行は件数右側の縦三点メニューから編集と削除を行い、既定リストでは削除を表示しない。
- リスト編集は編集領域からフォーカスが外れた時に自動保存し、空白の名称は編集前の名称へ戻す。
- 左ペインは `Ctrl+B` で開閉できる。
- 左ペインには今日が開始予定または期限のタスクを表示するビューを持つ。
- 左ペインにはお気に入りされたタスクを表示するビューを持つ。
- 中央ペイン上部でリスト、かんばん、カレンダーの表示形式を切り替える。タイムラインは実装と同時に追加する。
- 選択中のリスト、今日、お気に入りと表示形式は別の状態として扱う。
- アプリ上部右側にローカル検索と独立ポモドーロ画面への導線を表示する。
- 中央ペインには現在選択中のビューを表示する。通常作業の中心はタスク一覧ビューとする。
- かんばんは中央ペインで初期状態の `未着手`、`進行中` を表示し、状態列を追加、名称変更、並べ替え、削除できる。
- かんばんのタスクカードにはタイトル、期限、タグ、メモプレビュー、サブタスク進捗、タイマー実行中状態を表示する。
- かんばんからの状態変更と状態列の並べ替えは、Pointer/Keyboard対応のドラッグ&ドロップで行う。
- かんばんの状態追加は状態列の最右端、タスク追加は各状態列の未完了一覧上部に配置する。状態列から作成したタスクは同じトランザクションでその列へ所属させる。
- かんばん状態列の管理は見出し右端の縦三点メニューへ集約し、状態名編集、状態削除、確認付きの完了タスク全件削除、列内の表示ソートを行える。列上部の並べ替えハンドルは列幅全体を操作領域にする。
- 完了済みタスクは元の状態列に所属したまま、列下部の折りたたみ可能な完了セクションへ表示する。
- カレンダーは、中央ペインの表示形式として扱う。
- 現状のローカル通知設定は、左ペイン最下部の設定画面へ移管する。
- 親タスク作成は一覧の追加操作、かんばんの追加操作、カレンダーの空き領域操作、または `Ctrl+N` から共通ダイアログで開始する。
- 共通ダイアログは通常作成では期限、カレンダー作成では予定期間を初期値にし、保存成功時だけ閉じる。作成後に右詳細ペインは自動表示しない。
- 今日ビューからの通常作成は、最後に選択した実リストへ保存し、開始予定日をローカル日付の今日にする。
- サブタスク追加は親タスク詳細内のインラインフォームを維持する。
- タスクの完了操作は円形チェックボタンで行える。
- 完了済みタスクは訂正線と半透明で表示し、画面下部の完了セクションへ移動する。
- タスク行の右端にお気に入りボタンを表示する。
- タスク行には、必要に応じてサブタスク進捗、進捗バー、期限ありアイコン、タイマー実行中状態を表示する。
- 進捗バーは親タスクでありサブタスクが存在する場合だけ表示する。
- 右ペインは基本非表示とし、タスクまたはカレンダー項目を選択したときに詳細を表示する。
- 右ペインではサブタスク追加、期限日/期限時刻設定、繰り返し設定、通知予定表示、タイマー目標時間設定、タイマー操作、タスク削除を扱う。
- 右ペインの初期表示は現在情報の参照を主にし、編集フォームは必要な操作時だけ展開する。
- 親タスク詳細では既存サブタスクを一覧表示し、既存サブタスクの編集はサブタスク詳細で扱う。
- タスク一覧ではサブタスクを展開でき、サブタスク詳細から親タスク詳細へ戻れる。
- 左ペインやビューを切り替えた場合、右ペインの詳細表示は閉じる。
- タイマー操作は開始、一時停止、再開、終了のUIを持つ。
- 右ペインのタイマーセクションでは、通常タイマーとポモドーロを操作できる。
- ポモドーロ中は、右ペインで現在フェーズ、残り時間、状態、セット数を確認し、開始、一時停止、再開、作業完了、休憩開始、休憩完了、休憩スキップ、終了を操作できる。
- UI/UX詳細は [UI/UX再設計仕様](ui-ux-redesign.md) を正とする。

## 非機能要件

- アプリ実行時の外部通信は禁止。
- Windows/macOSで動作する。
- データはローカルSQLiteに保存する。
- インターネット接続なしで利用できる。
- ユーザー内容を端末外へ送信しない。
- 5,000件のタスクと20,000件のサブタスクでもUI操作が固まらないことを目指す。

## 受け入れ条件

- ユーザーがタスクとサブタスクを作成できる。
- タスクとサブタスクの期限日・期限時刻を設定できる。
- タスクタイマーを開始でき、停止するまで別タイマーを開始できない。
- サブタスクタイマーを開始でき、停止するまで別タイマーを開始できない。
- タイマー開始中にアプリを閉じて再起動しても、正しい経過時間を表示できる。
- カレンダーでタスクとサブタスクを確認できる。
- 未完了サブタスクがある親タスクは、確認後にだけ完了できる。
- タスク削除時に関連タイマー履歴がソフト削除される。
- 通知全体ON/OFFと通知表示モードを設定画面で選べる。
- ポモドーロ既定値を設定画面で変更できる。
- 右ペインで選択タスクまたはサブタスクのポモドーロを開始し、フェーズを操作できる。
- ポモドーロ作業/休憩の期限到達時に、フェーズが確定され通知設定に従って通知される。
- アプリ実行時にネットワーク通信しない。
- 左ペインを開閉でき、狭いウィンドウ幅でも主要操作が破綻しない。
- タスク一覧からお気に入り、進捗、期限有無を確認できる。
- かんばんで状態別にタスクを確認し、クリック操作で状態を変更できる。
- 右ペインで選択タスクの詳細編集とタイマー操作ができる。
- 今日ビューで、開始予定日または期限日が今日のタスクと、それらが今日のサブタスクを持つ親タスクを確認できる。
- カスタムリストを作成、名称変更、削除できる。
- 選択中リストで作成したタスクがそのリストに所属する。
- カスタムリスト削除時、所属タスクが初期リストへ移動し、タイマー履歴が削除されない。

## 決定事項

- 未完了サブタスクがある親タスクの完了時は確認メッセージを表示する。ユーザーが確認した場合、親タスクを完了できる。
- タスク削除時は、タスク、サブタスク、タイマー履歴、ポモドーロ履歴、通知ルールをソフト削除する。履歴があることを理由に削除をブロックしない。開始中タイマーやポモドーロもソフト削除し、通常のアクティブ検索から除外する。
- 通知表示は `title_only` をデフォルトとする。設定で通知全体OFFまたは `generic` に切り替えられる。
- カスタムリスト削除時はタスクを削除せず、初期リストへ移動する。
