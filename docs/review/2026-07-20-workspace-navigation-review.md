# ワークスペース再編 設計レビュー

対象: 親タスク作成ダイアログ、対象範囲/表示形式分離、検索、タスク色、詳細整理。

## 判断

分割実装を条件に承認する。

- 詳細整理、左タグ非表示、リスト色編集移動、かんばんblur保存は既存Use Case内で完結するため先行する。
- 左ペインからカンバン、カレンダー、ポモドーロを削除する変更は、中央切り替えと右上ポモドーロ導線を同時に実装する。
- 親タスク作成ダイアログは一覧とカレンダーのローカルstateをApp Shellへ統合する。
- 検索はApplication QueryとSQLite実装を追加し、読込済みページだけを対象にしない。
- タスク色はnullable継承モデル、DB移行、Read Model、エクスポートを同じ変更で実装する。
- タイムラインは読み取り専用の初回実装として独立Issueにする。

## 破綻シナリオ

- 代替導線より先に左ナビ項目を削除し、カンバンやカレンダーへ到達できなくなる。
- 表示形式切り替えで選択リストが既定リストへ戻る。
- ページング外のタスクが検索結果へ出ない。
- 遅い検索応答が新しい結果を上書きする。
- Enter送信後のblurでかんばん名が二重保存される。
- タスク色未設定を既定色で保存し、リスト色の継承が切れる。
- 作成ダイアログの送信中に二重作成される。

## スケール

- 検索は最大50件、200msデバウンス、120文字制限とし、大量データfixtureで計測する。
- 部分一致検索が遅くなった場合はSQLite FTSを代替案とするが、MVPで新しい同期テーブルを先行導入しない。
- 表示形式切り替えは同じQuery keyの再取得を避け、現在のRead Modelを再利用する。
- タイムラインは表示期間と行数を制限し、全履歴を一度にDOMへ展開しない。

## セキュリティ

- 検索Queryはバインド値とワイルドカードエスケープを使う。
- タイトル、メモ、タグ、検索語をHTML、ログ、外部通信へ渡さない。
- 色は許可済みトークンだけを受け入れ、任意CSSを禁止する。
- ダイアログ入力はPresentationだけでなくApplication層でも検証する。
- 新しいOS権限、ネットワーク権限、分析SDKを追加しない。

## 権限境界

- Presentationは画面stateと入力収集だけを担当する。
- Applicationは検索入力、作成入力、色トークン、対象IDを検証する。
- InfrastructureはSQLite検索と永続化を担当する。
- ポモドーロは既存の単一アクティブ制約と通知adapter境界を維持する。

## テスト計画

- Presentation: 左ナビ表示、中央切り替え、検索結果、ダイアログフォーカス、blur保存、詳細セクション非表示。
- Application/Infrastructure: ページング外検索、ワイルドカード、削除済み除外、タスク色検証、旧DB移行。
- Export: null継承と個別色のJSON/CSV。
- Performance: 大量データ検索、同一表示再選択、App/一覧の不要再描画。
- Manual: Tauri WebViewのダイアログ、検索キーボード操作、狭幅表示。

## テスト不足

- Tauri WebViewのフォーカストラップとIME入力はDOM単体テストだけでは保証できない。
- Windows実機の検索入力、ダイアログ、ポモドーロ導線は最終確認が必要である。

## Issue #161 実装後レビュー

- 判断: 承認。詳細、左ペイン、かんばんに限定し、DomainとSQLiteスキーマを変更していない。
- トランザクション境界: リスト名称と色は別々に保存せず、既存 `UpdateTaskList` へ1回で渡す。かんばん名称はEnterとblurを同じ保存関数へ接続し、保存中ガードで二重送信を防ぐ。
- データ保全: 左ペインのタグ表示だけを外し、タグデータ、Repository、詳細内の付与/管理は維持する。
- 到達性: カンバン、カレンダー、ポモドーロの左導線は、代替導線を実装するIssue #157まで維持する。
- スケール: 50タスク、200サブタスク、4リストのUIスモークで、リスト色保存50ms、かんばんblur保存43ms、詳細表示32ms、同一ナビ再選択19msを確認した。
- セキュリティ: 色は既存の許可済みトークンだけを扱い、外部通信、追加権限、ユーザー本文のログ出力を追加していない。実行時プライバシー監査39ファイルが成功した。
- 検証: TypeScript/Vite本番ビルド、Rust全106テスト、UIスモーク、README画像生成、差分空白検査が成功した。
- 残存リスク: Windows実機の左ペイン編集幅とIMEを使ったかんばん名称blur保存は、次回実機確認で確認する。
