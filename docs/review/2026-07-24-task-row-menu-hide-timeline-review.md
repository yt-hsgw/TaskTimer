# タスク行三点メニューとタイムライン一時非表示 レビュー

対象: GitHub #188 / docs/issues/074

## 判断

フォローアップ付き承認。タスク行の補助操作はPresentationのメニューに閉じ、永続順序の変更だけApplication Use Caseとして追加する。タイムラインはコード削除ではなく通常導線から隠す。

## チェックリスト

- 要件は docs/issues/074 に記録した。
- Clean Architectureの境界は、行順変更をApplication Use Case、SQLite実装をInfrastructureへ分離する。
- 期限、リスト、サブタスク入力は既存のDomain/Application検証を通す。
- 外部通信、追加権限、ユーザー本文ログは追加しない。
- タイムラインの保存済み設定はリストへフォールバックする。

## 破綻シナリオ

- メニューのクリックイベントが親行クリックへ伝播し、詳細ペインが開く。
- 上下移動で表示中のスマート範囲だけを並べ替え、所属リストの永続順序と矛盾する。
- 境界行の上/下移動で隣接タスクがなくエラーになる。
- タイムラインを最後の表示形式として保存していたユーザーが起動時に空画面を見る。

## スケール懸念

- メニューは表示中行だけに描画し、行ごとの常時ポップアップDOMを増やさない。
- 上下移動は隣接1件だけを検索し、全タスクの再採番を避ける。
- 更新後は既存のページ再同期境界を使い、ページング済み件数を保つ。

## セキュリティ懸念

- サブタスク名、期限入力、リストIDはApplication側で検証する。
- メニュー項目にユーザー入力HTMLを挿入しない。
- 削除は既存のソフト削除Use Caseを使う。

## テスト方針

- Rust: 行順変更Use Caseの正常系、上端/下端、削除済み・アーカイブ済み除外を確認する。
- Frontend: ビルドで型とイベント配線を確認する。
- UI計測: smoke profileでタイムライン非表示、タスク行メニュー、既存D&D/タイマーの回帰を確認する。
