# 設定方針

## 実行時設定

実行時設定はローカルのみとする。

許可する例:

- ウィンドウサイズと位置。
- テーマ。
- カレンダーの開始曜日。
- 通知の事前通知時間。
- 最後に選択した画面。

保存先:

- アプリ挙動に関わるユーザー設定はSQLiteを優先する。
- 表示専用の設定だけ、アプリ設定ファイルを検討する。

## 禁止する実行時設定

追加しないもの:

- APIキー。
- リモートエンドポイントURL。
- 分析識別子。
- リモートFeature Flagプロバイダー。
- ネットワークアクセスを有効化する隠しオプション。

## 環境ファイル

`.env` と `.env.*` は無視する。秘密情報をコミットしてはいけないため。

将来ビルドツールでサンプル値が必要になった場合は、非秘密のプレースホルダーだけを `.env.example` に追加する。

## Tauri権限の基準

実装は最小権限から始める。

- アプリデータディレクトリへのアクセス。
- ローカル通知権限。
- 将来ADRで承認されない限り、shell実行権限を追加しない。
- import/export機能で明示的に必要になるまで、任意ファイルシステムアクセスを追加しない。
- 実行時HTTP権限を追加しない。
