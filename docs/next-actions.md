# 次の作業

## 最優先

1. 親タスク完了時の確認ダイアログを実装する。
2. 削除時のソフト削除フローを実装する。
3. タスク/サブタスク編集を実装する。
4. 通知表示モード設定を保存できるようにする。
5. OS通知アダプターを実装する。

## 完了済み

- SQLite接続とマイグレーション実行器の初期実装。
- 週カレンダー、アクティブタイマー、通知表示設定のRepository境界。
- `CreateTask`、`CreateSubtask`、`StartTimer`、`StopActiveTimer` のApplication Use Case。
- タスク/サブタスク作成、タイマー開始/停止のSQLiteトランザクション実装。
- 単一アクティブタイマー制約と停止時 `elapsed_seconds` 確定のテスト。
- タスク一覧とタスク詳細の読み取りRepository。
- UIのモックデータ削除とSQLite由来表示への切り替え。
- タスク/サブタスク作成フォームとタイマー開始/停止ボタンのUse Case接続。
- 週カレンダーの日付範囲DBクエリ表示。
- remote未設定時のGitHub Issue草案。

## 次に必要

1. 親タスク完了時の確認ダイアログを実装する。
2. タスク削除で子サブタスク、タイマーセッション、通知ルールをソフト削除する。
3. サブタスク削除でタイマーセッション、通知ルールをソフト削除する。
4. タスク/サブタスク編集とステータス更新をUse Caseへ追加する。
5. 表示件数上限に到達した場合のUI表示を追加する。

## 通知まわり

1. 通知表示モード設定を保存できるようにする。
2. 通知本文生成で `title_only` と `generic` を切り替える。
3. OS通知アダプターを実装する。
4. 通知登録失敗時の `registration_status` と再試行を実装する。

## 実務運用前に必要

1. Windows/macOSの手動確認手順を実行する。
2. 実行時に外部通信していないことを確認する。
3. ログにタスク名・メモ本文・通知本文が出ないことを確認する。
4. GitHub上にリポジトリを作成し、Actionsのチェックを有効化する。
5. リリース前に署名・配布形式・インストール手順を決める。

## 危険ケース

- DB接続前にUI操作だけを作り込み、ドメインルールがUIへ漏れる。
- タイマー開始中の削除や親タスク完了で状態不整合が起きる。
- 通知登録失敗をUIで見せず、通知が来ない原因が分からなくなる。
- 開発時依存の通信と、アプリ実行時通信の禁止を混同する。
