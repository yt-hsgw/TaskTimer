# ADR 0003: 単一アクティブタイマー

## 状態

採用済み

## 背景

ユーザーは、同時に開始できるタスク/サブタスクタイマーを1件だけにする方針を確認した。

## 決定

`stopped_at IS NULL` の `timer_sessions` 行を1件だけ許可する。

## 理由

- 作業時間の曖昧さを避けられる。
- アクティブタイマーUIが単純になる。
- 経過時間の会計が明確になる。
- SQLiteのpartial unique indexで制約を強制できる。

## トレードオフ

- 並行作業を意図的に記録できない。
- 新しいタイマーを開始するには、現在のタイマーを停止する必要がある。

## 代替案

複数のアクティブタイマーを許可する。

利点:

- 重なった作業を表現できる。

欠点:

- レポートが難しくなる。
- UIが複雑になる。
- 誤って作業時間を水増しするリスクが上がる。
