# ADR 0002: オフライン優先ローカル保存

## 状態

提案中

## 背景

アプリは個人のタスク、メモ、予定、タイマー履歴を扱う。要件では外部通信を行わないことが求められている。

## 決定

ローカルSQLiteを唯一の正とする。MVPではクラウド同期、分析、クラッシュアップロード、リモートアセット、自動更新ネットワーク呼び出しを追加しない。

## 理由

- ユーザー内容を端末内に保てる。
- インターネットなしで動作する。
- プライバシーと権限境界が単純になる。
- トランザクション挙動が予測しやすい。

## トレードオフ

- 複数端末同期はできない。
- ローカルバックアップとエクスポートは、外部通信を使わない範囲で別ADRに従う。詳細は [ADR 0006](0006-local-backup-export-policy.md) に記録する。
- GitHubをアプリデータ同期先として使えない。

## 代替案

任意のクラウド同期を追加する。

利点:

- 複数端末で継続利用しやすい。
- ユーザーにとってバックアップが楽になる。

欠点:

- MVPの外部通信禁止要件に反する。
- 認証、競合解決、暗号化、プライバシー設計が必要になる。
